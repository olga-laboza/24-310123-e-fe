1. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?
	
	- в командной строке gitBash переходим в необходимую директорию и находясь в ней выполняем команду git init, 
	тем самы инициализирую репозиторий;
	- затем подготавливаем необходимые файлы для коммита командой git add <file name> или git add . 
	(если необходимо отправить все содержимое репозитория)
	- создаем коммит с помощью команды git commit -m "Comment for commit";
	- далее идем на github.com и создаем там репозиторий с таким же именем, как у локального, 
	при этом не создаем файл readme (не ставим галочку в соответствующем поле)
	- на открывшейся странице проверяем привязку по протоколу SSH и находим поле с заголовком " …or push an existing repository from the command line ",
	копируем следующее содержимое:
	git remote add origin git@github.com:olga-laboza/testtt.git - связывает локальный репозиторий с удаленным 
	(можно проверить командой git remote -v)
	git branch -M main - переименовывает ветку master в main
	git push -u origin main - отправляет содержимое локального репозитория на удаленный, с созданием ветки main
	- вставляем в командную строку gitBash и выполняем. Репозитории созданы и связаны.


2.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
	
	Отмена изменений файлов до внесения их в индекс осуществляется командой git checkout <file name> или git checkout .
	(если необходимо отказаться от всех изменений)


3.Как вернуть файл из индекса в рабочий каталог?
	
	Для возвращения файла из индекса используется команда git restore --staged <file name>
	

4.Как отменить ошибочный commit?

	Для отмены ошибочного коммита используется команда git reset HEAD~1, данная команда отменяет 1 коммит,
	если необходимо отменить больше, то после тильды ставится нужное число
